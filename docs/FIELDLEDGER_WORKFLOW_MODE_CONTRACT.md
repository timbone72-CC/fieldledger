# FieldLedger Workflow Mode Contract

## 1. Current Active Phase

Workflow Expansion Phase

Primary focus:

- Google Sheets usability
- CSV workflow improvement
- trusted-user workflow
- timesheet readability
- export usability
- operational friction reduction

## 2. Stable System Rule

Previously verified systems remain trusted unless related code changes.

Stable systems currently include:

- pay calculations
- JSON backup/import
- printable report
- mileage integration
- export flow
- onboarding/help wording
- mobile usability
- photo attachment system
- core summaries

These systems must not be repeatedly re-tested unless:

- related code changed
- bug reported
- regression suspected

## 3. Build Mode Rule

Build mode is the default operating mode.

In build mode:

- prioritize forward progress
- only test affected systems
- avoid reopening unrelated audits
- prioritize roadmap work

## 4. Verification Expiration Rule

Completed verification remains valid until related code changes.

## 5. Work-First Rule

If related systems did not change:

- continue building forward
- do not repeat walkthroughs
- do not repeat unrelated manual testing

## 6. Roadmap Priority Rule

Current roadmap priority:

- Google Sheets workflow
- export pipeline usability
- trusted-user operations
- real-world field workflow

These priorities override older unrelated audit findings.

## 7. Definition Of Done

A feature is complete after:

- implementation
- targeted verification
- integration verification for affected systems only
- stable commit
- checkpoint update if milestone-level

Completed features return to trusted baseline status.
